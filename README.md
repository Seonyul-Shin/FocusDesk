# FocusGuard (과제 집중 도우미)

노트북으로 과제를 하거나 자료를 찾을 때, 자신도 모르게 유튜브나 커뮤니티, 뉴스 탭을 열어 시간을 낭비하는 것을 방지하고 집중도를 기록할 수 있는 웹 애플리케이션 및 크롬 확장 프로그램 패키지입니다.

이 프로젝트는 별도의 무거운 빌드 과정(Node.js/NPM 등) 없이 브라우저에서 바로 실행하고 Vercel에 정적 페이지로 즉시 배포할 수 있는 초경량 구조로 설계되었습니다.

---

## 🏗️ 전체 아키텍처

1. **웹 대시보드 (`/web-app`)**:
   - 가입/로그인 및 집중 타이머 작동, 과제(태스크) 관리 및 사이트별 댓글(사용 목적) 작성, 공부용/딴짓용 분류 관리.
   - Vercel에 배포하거나 로컬에서 `index.html`을 여는 것으로 바로 작동합니다.
2. **크롬 확장 프로그램 (`/extension`)**:
   - 백그라운드에서 현재 활성화된 브라우저 탭의 체류 시간을 초 단위로 실시간 측정하여 Firebase에 기록합니다.
   - **용량 최적화**: 탭이 바뀌거나 15초 단위로 데이터를 일괄 전송(Batching)하여 Firebase의 대역폭과 트래픽 소모를 최소화합니다.

---

## 🛠️ 설정 및 사용법

### 1단계: Firebase 설정하기
앱이 데이터를 실시간으로 보관하기 위해 Firebase Realtime Database와 Authentication이 필요합니다.

1. [Firebase 콘솔](https://console.firebase.google.com/)에 로그인하고 **프로젝트 추가**를 클릭합니다.
2. 빌드 메뉴에서 **Authentication**으로 이동하여 **이메일/비밀번호** 로그인 방식을 활성화합니다.
3. 빌드 메뉴에서 **Realtime Database**를 생성합니다. (데이터베이스 위치는 가까운 국가로 설정)
4. 생성 후, **규칙(Rules)** 탭으로 이동하여 읽기/쓰기 권한이 사용자 본인에게만 허용되도록 설정합니다:
   ```json
   {
     "rules": {
       "users": {
         "$uid": {
           ".read": "$uid === auth.uid",
           ".write": "$uid === auth.uid"
         }
       }
     }
   }
   ```
5. 프로젝트 개요 페이지의 톱니바퀴 아이콘(프로젝트 설정) -> 앱 추가 -> **웹(</>)** 아이콘을 클릭하여 웹 앱을 등록합니다.
6. 제공되는 Firebase 구성(Configuration) 객체 정보(API Key, DB URL 등)를 확인합니다.

---

### 2단계: 웹 앱 구성 설정
1. `/web-app/config.js` 파일을 엽니다.
2. `firebaseConfig` 객체 안에 1단계에서 획득한 Firebase 접속 정보들을 입력하고 저장합니다:
   ```javascript
   export const firebaseConfig = {
     apiKey: "본인의_API_KEY",
     authDomain: "본인의_PROJECT_ID.firebaseapp.com",
     databaseURL: "https://본인의_PROJECT_ID-default-rtdb.firebaseio.com",
     projectId: "본인의_PROJECT_ID",
     storageBucket: "본인의_PROJECT_ID.appspot.com",
     messagingSenderId: "본인의_SENDER_ID",
     appId: "본인의_APP_ID"
   };
   ```

---

### 3단계: 크롬 확장 프로그램 로드하기
크롬 브라우저에서 사용자가 여는 다른 탭을 추적하기 위해 개발자 모드로 확장을 추가해야 합니다.

1. 크롬 브라우저를 열고 주소창에 `chrome://extensions`를 입력해 이동합니다.
2. 우측 상단의 **개발자 모드(Developer mode)** 스위치를 활성화합니다.
3. 좌측 상단에 나타나는 **[압축해제된 확장 프로그램을 로드합니다]** (Load unpacked) 버튼을 누릅니다.
4. 이 레포지토리의 `/extension` 폴더를 선택하여 로드합니다.
5. 크롬 주소창 옆의 퍼즐 모양 아이콘을 클릭하고 **FocusGuard Companion**을 고정(Pin)합니다.

---

### 4단계: 동기화(Sync) 연동하기
1. `/web-app/index.html` 파일을 더블 클릭하여 웹 앱을 실행합니다 (또는 Vercel에 배포한 주소로 접속).
2. 회원가입을 완료한 뒤 로그인합니다.
3. 웹 화면 우측 하단의 **[연동 방법 보기]** 또는 사이드바 하단 버튼을 클릭하면 **내 연동 Sync Key (UID)**가 표시됩니다. 이를 복사합니다.
4. 크롬 툴바의 **FocusGuard Companion** 확장 아이콘을 클릭합니다.
5. 복사한 **Sync Key**와 본인의 **Firebase Database URL**을 입력한 뒤 **[연동 및 설정 저장]** 버튼을 클릭합니다.
6. 연동 상태가 **"연동됨"**으로 바뀌면 모든 준비가 끝났습니다!

## 🚀 서비스 이용 시나리오

1. **과제 추가**: 대시보드 왼쪽 상단의 `+ 추가` 버튼을 눌러 과제명(예: `알고리즘 과제 1`)을 입력합니다.
2. **타이머 작동**: 생성된 과제를 선택하고 `시작`을 누릅니다. 대시보드 타이머가 작동하며 확장 프로그램이 활성화 상태로 전환됩니다.
3. **서핑 & 자료조사**: 이제 평소대로 구글링을 하거나 유튜브를 시청합니다. 타이머가 켜져 있는 동안 이동한 모든 사이트 체류 시간이 백그라운드에서 정밀하게 측정됩니다.
4. **목적 기록**: 방문한 사이트 카드가 대시보드에 실시간으로 표시됩니다. 해당 사이트 링크 카드 하단의 입력창에 왜 들어갔는지 목적(댓글)을 기입합니다. (예: "퀵정렬 개념 블로그 참고", "머리 식히려고 본 유튜브 예능")
5. **공부용/딴짓 분류**: 각 사이트 카드에서 `과제용` 또는 `딴짓용` 버튼을 누르면 카드가 유용한 사이트와 딴짓 사이트 컬럼으로 이동하며 집중도가 자동 집계됩니다.
6. **집중도 확인**: 대시보드 중앙의 원형 그래프를 통해 과제 시간 대비 딴짓으로 낭비한 비율이 몇 %인지 실시간 시각적 분석 통계를 확인합니다.
7. **기록 정지**: 과제가 끝나면 `중지` 버튼을 누릅니다. 타이머가 대기 상태로 전환되며 탭 추적도 즉시 종료됩니다.

---

## ⚡ Firebase 비용 및 트래픽 절약 설계
- **초경량 데이터 스키마**: 데이터 노드 키 길이를 1~3자(`t`, `dur`, `cat`, `cmt` 등)로 극도로 단축하여 Firebase Realtime Database의 스토리지 점유율과 다운로드 대역폭 요금을 최소화했습니다.
- **REST API fetch 동기화**: 크롬 확장 프로그램은 무거운 Firebase 클라이언트 SDK 라이브러리를 로드하지 않고, 가벼운 브라우저 네이티브 `fetch` API를 사용하여 백그라운드 리소스와 요금 부하를 최소화합니다.
- **체류 시간 집계 최적화**: 1초마다 실시간 쓰기를 수행하지 않고, 사용자가 탭을 전환하거나 브라우저 비활성화가 감지될 때 또는 **15초 단위 정기 동기화 알람**이 올 때에만 누적된 시간을 Firebase로 일괄 전송합니다.
